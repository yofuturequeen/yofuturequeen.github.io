Final Prototype Writeup

Overview and Usage

This game is my Strawberraroll Prototype. This is a prototype for my final project which means while it is functional, it doesn't have all the bells and whistles my final version will have. Since it builds off my original game, the prototype version contains some new functionality. You can play my game prototype here.

Issue Deep-Dive

The biggest problem I ran into for this prototype version was getting all the functionality to work upon ending the game. An issue here was my animation only playing for a split second when the poison was touched. This is because the draw loop would continue meaning my animation would only play for one frame. I also tried to remove all sprites with end condition which didn't work properly either. This was a logic error and to fix this I created entirely different screens depending on which stage the user was on (start, in game, game over).

Further Questions

Project Scope

For my final project I added the idea of adding Firebase to my game to advance my game and make it a bit more complex. I also added different screens depending on what stage of the game I'm on which is a new change for my final. However, I don't think I will be able to add different characters unless I have ample time after completing all my other changes.

Time Management

While I didn't neccessarily predict which issues I would run into, I knew I would run into a lot. I anticipated getting stuck quite often and that's exactly what happened. It felt like even making the smallest of changes would then turn me to a whole new set of problems which was really frustrating. Getting stuck on one small line of code for an entire work session definitely added more time to my completion as well.

Implementation Plan

For my final version, besides the changes I listed above, I will also run my game on my web dev server much more often. One part of this process I found quite time consuming is debugging my deployments so I will be sure to run that more often and earlier.
